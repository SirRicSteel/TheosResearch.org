------------------------------------------------------------
1. PURPOSE
------------------------------------------------------------

This section defines how contradiction between engines is transformed
into signal, and how signal is compressed into accumulated wisdom
without amplifying noise, bias, or hallucination.

------------------------------------------------------------
2. CONTRADICTION IS MANDATORY
------------------------------------------------------------

Agreement alone does not produce wisdom.

Wisdom emerges only when:
- Engines disagree
- Assumptions conflict
- Reasoning paths diverge
- Confidence margins compress

------------------------------------------------------------
3. TYPES OF CONTRADICTION
------------------------------------------------------------

THEOS recognizes:

- Structural contradiction (reasoning order)
- Assumptive contradiction (starting premises)
- Interpretive contradiction (meaning assignment)
- Outcome contradiction (recommended action)

All are valid inputs.

------------------------------------------------------------
4. SIGNAL VS NOISE DISTINCTION
------------------------------------------------------------

Noise:
- Random variation
- Stylistic difference
- Token-level instability
- Temperature artifacts

Signal:
- Persistent disagreement
- Coherent alternative explanations
- Stable divergence across cycles
- Reproducible conflict under similar inputs

------------------------------------------------------------
5. COMPRESSION MECHANISM
------------------------------------------------------------

Compression does NOT average outputs.

Compression:
- Identifies invariant overlap
- Isolates irreducible disagreement
- Preserves minority reasoning paths
- Removes redundant explanation mass

------------------------------------------------------------
6. ROLE OF THE GOVERNOR
------------------------------------------------------------

The Governor evaluates:

- Disagreement magnitude
- Improvement slope across cycles
- Energy cost vs refinement gain
- Risk introduced by resolution

The Governor decides:
- Continue cycles
- Swap engine roles
- Disengage one engine
- Freeze output
- Refuse answer

------------------------------------------------------------
7. NON-DESTRUCTIVE COMPRESSION
------------------------------------------------------------

No reasoning path is deleted.

Compressed wisdom retains:
- Why alternatives failed
- What assumptions were rejected
- What risks were avoided

------------------------------------------------------------
8. TEMPORAL DEPTH
------------------------------------------------------------

Compression introduces time.

Later cycles reason about:
- Earlier failures
- Prior contradictions
- Consequences of resolution

This creates memory with context, not recall.

------------------------------------------------------------
9. WISDOM STORAGE FORMAT
------------------------------------------------------------

Wisdom is stored as:

- Decision traces
- Constraint boundaries
- Failure envelopes
- Confidence gradients

Never as slogans or rules.

------------------------------------------------------------
10. NO FINAL ANSWER FALLACY
------------------------------------------------------------

Wisdom is not certainty.

A refined answer may still be:
- Probabilistic
- Contextual
- Time-bound
- Reversible

------------------------------------------------------------
11. CROSS-QUERY GENERALIZATION
------------------------------------------------------------

Wisdom may generalize only when:
- Contradictions recur
- Outcomes validate
- Context similarity exceeds threshold

Otherwise, wisdom remains local.

------------------------------------------------------------
12. FAILURE MODE PROTECTION
------------------------------------------------------------

If contradiction collapses prematurely:
- Governor forces re-expansion
- Engines are role-swapped
- Cycle depth increases

------------------------------------------------------------
13. ANTI-DOGMA CLAUSE
------------------------------------------------------------

Wisdom cannot harden into doctrine.

If contradiction ceases entirely:
- New perturbations are introduced
- Alternative framings are injected

------------------------------------------------------------
14. ENERGY EFFICIENCY
------------------------------------------------------------

Compression reduces:
- Token usage
- Redundant reasoning
- Repeated failure paths

This is how THEOS achieves higher quality
with lower compute.

------------------------------------------------------------
15. IMMUTABILITY CLAUSE
------------------------------------------------------------

The following are immutable:

- Contradiction is required
- Compression is selective, not averaging
- Wisdom preserves failure context
- Governor arbitrates compression
- No wisdom without disagreement
